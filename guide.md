# 🏠 SH 행복주택 서류심사 커트라인 당첨 예측기 MVP 프로젝트 가이드 (최종 완성판)

> **이 문서는 AI나 개발 지식이 없는 기획자부터 실제 시스템을 구현할 AI 엔지니어까지 프로젝트의 데이터 처리 원리, 수집된 원본 데이터의 가공 필터링 정책, 그리고 머신러닝 구동 원리를 완벽하게 이해할 수 있도록 작성되었습니다.**

---

## 1. 💡 쉬운 비유로 이해하는 프로젝트 목적

이 프로젝트는 **"내가 가진 조건(나이, 거주지, 자산 등)으로 행복주택 특정 단지에 지원했을 때, 서류심사를 통과할 수 있을까? 확률은 얼마나 될까?"**를 알려주는 시스템입니다.

이 시스템이 돌아가는 원리는 **'대학 입시 합격 예측기'**와 완전히 똑같습니다.

```
1. [모집 요강] ➔ 입주자 모집 공고문 (PDF): 나이, 거주지, 청약통장 납입 횟수 등에 따라 점수를 주는 규칙집
2. [배치표 (과거 컷트라인)] ➔ 서류심사 커트라인 (PNG 이미지): 과거에 이 아파트에 들어가려면 최소 몇 점이어야 합격했는지 적힌 표
3. [수험생 성적표] ➔ 신청자 프로필 (Java DTO): 사용자가 입력한 나이, 자산, 소득 등의 원천 정보
4. [합격 예측기] ➔ 예측 엔진 (AI): "작년 강남구 아파트 컷트라인이 6점이었는데, 당신의 성적(조건)은 6점이므로 합격 확률은 약 50%(동점자 추첨)입니다"라고 계산해 주는 역할
```

## 2. 📂 어떤 데이터를 사용했고 무엇을 버렸는가? (데이터 필터링 정책)

프로젝트에는 사용자로부터 받은 **2가지 원본 데이터셋**이 존재합니다. 인공지능 학습을 위해 유의미한 항목만 추출하고 노이즈 데이터는 전부 버렸습니다.

### ① 규칙 데이터 (PDF 모집 공고문)
- **원본**: `rentalhome_rule/rule.pdf` (2025년 2차 행복주택 입주자 모집 공고문 - 총 15페이지)
- **살린 데이터 (사용한 항목)**:
  - **신청 계층 구분**: 대학생, 청년, 신혼부부, 고령자, 주거급여수급자의 자격 조건
  - **자격 제한 기준**: 계층별 소득 한도(%), 총 자산 한도, 자동차 소유 및 가액 제한 기준
  - **순위 산정 기준**: 거주지/소득 근거지에 따른 1순위, 2순위, 3순위 판정 기준
  - **가점 표**: 서울시 거주기간(1~3점), 청약통장 납입 횟수(1~3점), 나이 및 취약계층 가산점(1~3점) 등 가점 연산 공식
- **버린 데이터 (제외한 항목)**:
  - 주택 공급 현황 상세 표 (단지별 동호수 배치 등 모델 예측과 무관한 항목)
  - 청약 신청 일정, 신청 방법 안내 텍스트, 제출 서류 목록 등 행정 안내 사항
  - 입주자 유의사항, 계약 체결 안내, 예비입주자 중복선정 제한 등 법적 고지 문구

### ② 커트라인 데이터 (PNG 이미지 표)
- **원본**: `rentalhome_cutline` 폴더 내 PNG 파일들 (2021년 제1차 행복주택 서류심사대상자 커트라인 공고 문서 - 총 99페이지)
- **살린 데이터 (사용한 항목)**:
  - **단지명 / 주택 면적(㎡) / 공급 유형 (대학생, 청년 등) / 공급 방식 (우선, 일반)**
  - **합격선 순위 및 가점**: 예) "1순위 6점"에서 `순위=1`, `가점=6`을 분리 추출
  - **동점자 판정 기준**: 예) "(자치구 전입일 : 2010.01.25)"에서 `기준=전입일자`, `값=2010.01.25`로 분리 추출
- **버린 데이터 (제외한 항목)**:
  - 표의 헤더 장식선, 로고 및 페이지 번호 이미지 데이터
  - 커트라인 표 우측의 '비고'란 및 안내 코멘트
  - 커트라인 기준에 도달하지 못해 발생한 무의미한 기호나 공백

## 4. 🔍 데이터 변환의 실제 예시: 이미지 한 칸에서 AI 데이터셋까지

이미지 표 속에 존재하는 복잡한 텍스트 한 줄이 어떻게 데이터화되고 인공지능이 학습할 수 있는 형태로 변환되는지 단계별 예시로 설명합니다.

### 1단계: 이미지 속 원본 데이터 (Raw Text)
2021년 1차 커트라인 표의 특정 행에 아래와 같은 텍스트가 있습니다.
> 디에이치포레센트(일원대우) | 59 | 행복주택(신혼부부계층) | 우선 | 1순위 6점(자치구 전입일 : 2010.01.25)

### 2단계: 구조화된 데이터 정형화
인공지능(Gemini)이 위 텍스트를 읽고 기계가 이해할 수 있는 속성값(`parsed_cutlines.json`)으로 쪼갭니다.
```json
{
  "complex_name": "디에이치포레센트(일원대우)",
  "district": "강남구",
  "housing_type": "59",
  "supply_target": "newlyweds",
  "supply_type": "우선",
  "cutline_rank": 1,
  "cutline_score": 6,
  "tie_breaker": "전입일자",
  "tie_breaker_value": "2010.01.25"
}
```

### 3단계: 모의 지원자 데이터 매칭 및 라벨링 (Target 0 또는 1 생성)
실제 지원자 데이터를 구할 수 없으므로, 시스템이 가상의 지원자(모의 데이터)를 무작위로 생성하여 이 커트라인과 대조해 당첨 여부 정답을 매깁니다.

*   **가상 지원자 A 프로필**: 
    - 무주택자, 소득 80%, 강남구 거주, 청약통장 36회 납입, **강남구 전입일: 2008.05.12**
    - **가점 계산 결과**: 1순위 6점 (거주기간 3점 + 청약통장 3점)
    - **커트라인 대조**: 커트라인(1순위 6점, 전입일 2010.01.25 이전)과 비교 시, 지원자의 전입일(2008.05.12)이 기준일보다 빠르므로 합격.
    - **➔ 데이터셋 기록**: `[지원자 A 속성값들...]` ➔ **Target: 1 (합격)**

*   **가상 지원자 B 프로필**:
    - 무주택자, 소득 85%, 강남구 거주, 청약통장 36회 납입, **강남구 전입일: 2012.08.30**
    - **가점 계산 결과**: 1순위 6점 (거주기간 3점 + 청약통장 3점)
    - **커트라인 대조**: 순위와 점수는 컷트라인과 같으나, 전입일(2012.08.30)이 기준일(2010.01.25)보다 늦으므로 동점자 처리에서 탈락.
    - **➔ 데이터셋 기록**: `[지원자 B 속성값들...]` ➔ **Target: 0 (탈락)**

*   **가상 지원자 C 프로필**:
    - 무주택자, 소득 70%, 마포구 거주, 청약통장 24회 납입
    - **가점 계산 결과**: 우선공급 **2순위** (타자치구 거주자이므로 2순위 부여됨)
    - **커트라인 대조**: 커트라인은 1순위인데 지원자는 2순위이므로 미달 탈락.
    - **➔ 데이터셋 기록**: `[지원자 C 속성값들...]` ➔ **Target: 0 (탈락)**

## 5. 🧠 머신러닝 모델의 이해 및 성능 평가

최종적으로 학습 및 검증에 사용된 데이터셋은 이와 같이 매칭된 **4,840개의 가상 지원자 레코드**입니다.

### ① 입력 피처(Features)와 예측 대상(Target)
인공지능 모델이 들여다보는 항목들입니다.
- **입력 피처(X)**: `나이`, `혼인기간`, `소득비율`, `자산가치`, `차량가치`, `청약통장 납입 횟수`, `거주지`, `지원단지`, `지원 주택형`, `공급 구분(우선/일반)`
- **예측 대상(y)**: `Passed (당첨 여부: 0 또는 1)`

### ② 모델 평가 결과 (클래스 가중치 밸런싱 적용)
클래스 불균형(합격자 비율 7% 미만) 문제를 해결하기 위해 합격 데이터에 가중치(`class_weight='balanced'`)를 부여하여 고르게 훈련시킨 결과입니다.

| 테스트한 모델명 | 정확도 (Accuracy) | 정밀도 (Precision) | 재현율 (Recall) | F1-Score | ROC-AUC |
| :--- | :---: | :---: | :---: | :---: | :--- |
| Logistic Regression | 67.36% | 0.1357 | 66.67% | 0.2255 | 0.7211 |
| Decision Tree | 93.08% | 0.5172 | 0.4348% | 0.4724 | 0.7018 |
| Random Forest | 93.70% | 0.6538 | 0.2464% | 0.3579 | 0.9568 |
| **LightGBM** | **93.39%** | **0.5258** | **0.7391%** | **0.6145** | **0.9583** |

- **재현율(Recall)**: 실제 합격한 사람들을 AI가 놓치지 않고 얼마나 "합격할 것"이라고 잘 맞췄는가에 대한 비율입니다. (LightGBM이 73.91%로 가장 우수)
- **분별력(ROC-AUC)**: 합격자와 탈락자를 헷갈리지 않고 완전히 가려내는 능력으로 **0.9583**은 거의 완벽한 구분을 뜻합니다.

## 6. 🚀 상용화(Production) 서비스를 위한 데이터 확장 및 고도화 방향

## 7. 🏗️ 추천하는 서비스 아키텍처: "하이브리드 엔진"

실제 사용자 대상 상용 서비스를 배포할 때는 인공지능 단독 모델만 쓰는 것이 아니라, **"코딩(규칙 연산) + 인공지능(확률 보정)"**을 결합한 하이브리드 형태로 구성하는 것이 정석입니다.

```
[사용자 조건 입력]
       ➔ 1단계 (룰 엔진) : "당신의 자격 요건은 Pass이며 가점은 1순위 6점입니다" (정확도 100%)
       ➔ 2단계 (DB 대조)  : "원하시는 강남구 아파트의 과거 커트라인은 1순위 6점이었습니다" (정확도 100%)
       ➔ 3단계 (AI 모델) : "1순위 6점 동점자들 사이에서 과거 추첨 이력과 자치구 경쟁 강도를 분석했을 때, 
                           최종 당첨될 예상 확률은 34.2%입니다" (확률 제시)
```

- **룰 계산 코드(Java/Python)**: 사용자의 나이, 자산, 거주기간 등을 모집요강 공식 그대로 한 치의 오차도 없이 계산하여 100% 신뢰할 수 있는 정확한 점수를 띄워줍니다. (오차 0%)
- **인공지능 모델(LightGBM)**: 컷트라인 점수가 똑같아서 결국 **추첨을 해야 하는 애매한 상황**이거나, **과거 커트라인 정보가 없는 새로운 단지**를 지원할 때, 인공지능이 동점자들 사이의 예상 실질 당첨 확률(%)을 정교하게 보정해 주는 역할을 담당합니다.
